# Ga-Sheet-Patcher Architecture

Ga-Sheet-Patcher is a Google Apps Script library designed to efficiently synchronize a Google Sheet with a 2D array of data. It minimizes API interaction—which is crucial for Google Apps Script performance—by calculating precise structural changes (row/column reordering, insertion, deletion) and updating only the modified cell data.

## Core Concepts

The library operates on the principle of **state transformation**. Instead of blindly overwriting the entire sheet, it calculates the minimum set of operations required to transform the current sheet state to match the target `newData` state.

## Architecture

The library is organized into three main layers:

### 1. Structural Logic (`SeriesPatcher`)
The `SeriesPatcher` class is the algorithm engine. It is domain-agnostic; it simply calculates the `insert`, `delete`, and `move` actions required to transform one array of unique identifiers into another.

*   **Algorithm:**
    1.  **Deletions**: Identifies elements in the source array not present in the target array and generates `delete` actions (processing back-to-front).
    2.  **Insertions/Moves**: Iterates through the target array. For each element:
        *   If it matches the element at the same index in the working array, it skips.
        *   If it's missing, it triggers an `insert`.
        *   If it's elsewhere in the array, it triggers a `move`.

### 2. Sheet Orchestration (`SheetPatcher`)
`SheetPatcher` orchestrates the entire synchronization process.

*   **Workflow:**
    *   **Structural Alignment**: Uses `SeriesPatcher` to determine how to restructure rows and columns, then delegates to `RowAlignmentApplier` and `ColumnAlignmentApplier` to physically change the sheet structure.
    2.  **Minimal Data Patching**: Once the structure is aligned, it compares existing cell data with the new data.
    3.  **Contiguous Patching**: Within rows, it identifies contiguous chunks of cells that have changed and performs a single API call per chunk, rather than one call per cell.
    4.  **Verification & Fallback**: After patching, it verifies the sheet content matches the input. If it fails, it falls back to a full overwrite as a safety measure.

### 3. Structural Appliers (`AlignmentApplier` + Subclasses)
`AlignmentApplier`, `ColumnAlignmentApplier`, and `RowAlignmentApplier` implement the concrete Google Apps Script operations required to restructure the sheet based on the actions generated by `SeriesPatcher`.

## Design Choices

### Structural Alignment Order
The library performs row alignment before column alignment in `SheetPatcher`. This provides a stable base by finalizing the vertical structure (records) before adjusting the horizontal structure (attributes/headers), ensuring consistent identification anchors for both operations.

## Key Performance Considerations
- **Minimizing API Calls**: Google Apps Script's `SpreadsheetApp` API calls are expensive. The library mitigates this by:
    - Batching cell updates using `setValues` for contiguous changed blocks.
    - Using structural alignment to only update what is necessary.
- **Structural Integrity**: The library assumes the first row contains headers and the first column contains row identifiers. These are used as the "anchor" for reordering, insertion, and deletion operations.
